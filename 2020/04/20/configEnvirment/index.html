    重装系统

    有相当一段时间没有使用过mac了，由于数据同步，Matlab等等很多东西使用的不便，最近几个月都是在win上办事,但是LOL, Seven Days To Die, Fullout 76, CS:GO …诱惑实在是太大了,学习学着学着就快乐去了，在考研之前决定不再用Windows了，正好学习完全的使用mac办公。但是迁移博客时惊讶的发现windows上无法使用git，刚开始以为是自己新配置的代理的问题，更改了npm和git的代理后无果，后来发现mac也一样无法git clone,感觉是自己搞砸了，brew的配置当时也没有记录，很多软件都是瞎装，很心烦就重装了系统。重装之后问题依然无法解决，找资料后发现2020.2左右Github被墙了。

    但是在3月左右的时候上传代码依然是可行的，所以应该是之前配置过的东西发生更改了，就想到了GoogleHost，项目在一个月前更新过，替换原有的host文件后解决了问题。再记录一下重装系统后配置环境的过程。

    论文

    • 安装MacTex

    • 很多人认为mac系统的存储非常珍贵，MacTex太过庞大了，会用brew cask install basictex但是如果需要像beamer这类有大量依赖包的模版，还是使用mactex轻松简单的，不需要看着error 不停的去找对应的包。

    • 使用TexPad作为编辑器

      • TexPad 就像是Latex版的Typora一样，支持时时预览（左右对照版本的，cmd + t查看）。

    • 使用mac自带的==查询与数据检索器==流畅的阅读英文论文。

      • 在设置 - 触控板 - 光标与点按 的第一项设置查询与数据检索器设置可查看如何使用，三指轻点是我自己的习惯。真的超级棒，但是有的单词字典里没有，也可以直接在检索框里百度搜索。

      • 当然 com+space 的聚焦搜索也可以很快的查询单词，输入之后，com+L 就会快速的定位到单词搜索界面。
      • 不能读单词？在辅助功能的语音设置，按下option+esc就可以朗读文本啦，不过只会在聚焦里读一个词，别的地方会读一堆。。
      • 专业的字典柯林斯双解、朗文5for macos。

    • 英文划句翻译代替方案：Bob

      • option + D之后就会获得和Qtranslate相同的效果。

    • 使用mathpix snipping作为辅助公式工具：

      • mathpix 是使用OCR将数学公式翻译成Latex或mathtype的工具。
      • 支持win和macOS

    Typora 0.9.85 for windows无疑是最令人激动的更新了，自动图床不再是macos的专权，通过集成PicGo APP 、PicGo(command line),Typora 可以在你使用 win+shift+s和Ctrl+v 的同时，将图片上传到设置好的图床并生成链接，不用再托到PicGo，复制markdown再黏黏了，也不用安装PicGo的客户端，界面实在太丑了。Command line版可以在需要用到它的时候启动，再自动关闭，节省了系统资源。至此可以彻底告别SublimeText3中的垃圾Markdow插件了。

    问题

    配置完成后，终端测试正常：

    Typora Test 错误：

    但是在Typora中，将图片拖入后，右键上传，上传成功。

    已提交issue#3233。

    已解决，原因在于Github仓库不能出现同名文件。这个Issue在PicGo的项目中有过提交，我忽略了自己曾经使用客户端版本的PicGo测试过上传图片，所以在命令行下再次上传测试图片时，就会报错。这个报错是属于Git的，应该耐心的去看说明文档才对。

    从文档来看无论是设置了自动上传还是不设置都需要右键上传，全局自动上传会配置YAML，担心会和Hexo的配置出现冲突。所以只要配置优先使用相对路径，再上传就可以解决这些问题。

    Onedrive卸载后的残留问题

    由于Onedrive的账号失效（组被封禁），需要重注册一个5TB的云盘，但是老账户无法登陆，所以无法获得本地的Onedrive文件夹权限（无法取消关联）。所以卸载了Onedrive，但是本地的文件夹并没有消失，也无法删除。在网上查了很多资料无果后，尝试在管理员模式下：

    rd/s/q “Onedrive - Lo”

    在重分析点缓存区标记无效

    在Microsoft Support上找到解决方法：

    在驱动器（Onedrive文件夹所在的磁盘）工具->修复驱动器，重启之后即可直接删去。
